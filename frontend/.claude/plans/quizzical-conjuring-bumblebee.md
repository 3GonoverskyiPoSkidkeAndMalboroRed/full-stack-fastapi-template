# План: UX-доработки (автологин, чекаут, наличие, навигация админа, спиннеры)

## Контекст

Пять независимых UX-улучшений магазина, чтобы поведение было ожидаемым и отзывчивым:

1. **Автологин при регистрации** — сейчас после регистрации редирект на `/login`, пользователь логинится вручную. Нужно входить автоматически.
2. **Автозаполнение ФИО в чекауте из профиля** — функционал уже есть в коде; нужно проверить и починить, если не работает.
3. **Скрыть товары не в наличии** (`stock == 0`) из общего каталога.
4. **Упростить навигацию для админа** — оставить только «Каталог» и «Админка», кнопка «Кабинет» у админа ведёт в `/admin`.
5. **Спиннеры загрузки** — глобальный индикатор при переходах между страницами + спиннеры на кнопках, отправляющих данные.

Согласовано с пользователем: навигация админа — только Каталог + Админка (+ выход); чекаут — проверить и починить; спиннер — глобальный при переходах + на кнопках.

## Проверка (end-to-end)

Стек уже поднят (`docker compose`), правки фронта подхватываются билдом, бэкенд — пересборкой.

1. **Бэкенд:** `cd backend && uv run mypy app && uv run ruff check app` (фильтр stock). Пересобрать backend (`docker compose up -d --build backend`) и проверить API: `GET /api/v1/items/public` не возвращает товары с `stock==0`; деталь `GET /api/v1/items/public/{id}` для такого товара по-прежнему 200.
2. **Фронтенд:** `cd frontend && npm run lint && npm run build`.
3. **Ручная проверка в браузере** (`dashboard.localhost:8081`):
   - Регистрация нового пользователя → сразу залогинен, попадает на главную (не на `/login`).
   - `/checkout` у пользователя с заполненным профилем → ФИО/телефон/адрес подставлены.
   - В каталоге нет товаров «Нет в наличии» (для теста выставить `stock=0` товару в админке).
   - Вход под админом (`admin@example.com`): в шапке нет «Корзины», «Кабинет» ведёт в `/admin`; в сайдбаре только «Каталог» и «Админка».
   - На медленной сети (DevTools throttling) при переходах виден спиннер; кнопки «Оформить заказ» и «В корзину» показывают спиннер при отправке.
