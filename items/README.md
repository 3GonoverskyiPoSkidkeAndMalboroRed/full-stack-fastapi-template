# Каталог товаров для загрузки в БД

Эта директория — источник данных для seed-скрипта `backend/app/seed_items.py`,
который создаёт товары (`Item`) с фотографиями.

## Структура

Каждая поддиректория — один товар:

```
items/
└── <Бренд>_<Модель>/
    ├── item.json        # параметры товара
    ├── photo1.webp      # фотографии (.webp / .jpg / .jpeg / .png)
    ├── photo2.webp
    └── ...
```

Скрипт берёт **все** папки, в которых есть `item.json`. Папка без `item.json`
пропускается. Имя папки как данные не используется — всё берётся из `item.json`.

## Формат `item.json`

```json
{
  "title": "Givenhed Sunglasses",
  "brand": "Chrome Hearts",
  "category": "Аксессуары",
  "size_name": "One Size",
  "description": "Краткое описание (≤ 255 символов).",
  "cost": "999000.00",
  "stock": 1,
  "images": ["83.webp", "CHG2.webp", "CHG3.webp"]
}
```

| Поле          | Тип            | Описание                                                            |
| ------------- | -------------- | ------------------------------------------------------------------- |
| `title`       | строка         | Название товара. Используется для проверки дубликатов.              |
| `brand`       | строка         | Бренд. Создаётся, если ещё нет в БД.                                |
| `category`    | строка         | Категория. Создаётся, если ещё нет в БД.                            |
| `size_name`   | строка / null  | Размер. Создаётся, если ещё нет. Можно опустить.                    |
| `description` | строка / null  | Описание, до 255 символов.                                          |
| `cost`        | строка         | Цена. **≤ 999999.99** (ограничение колонки `Numeric(8,2)`).         |
| `stock`       | целое          | Остаток на складе.                                                  |
| `images`      | список строк   | Имена файлов из этой же папки. Порядок = порядок в галерее (1-й — главный). |

## Запуск

Скрипт работает с БД и статикой внутри контейнера `backend`, поэтому сначала
копируем папку `items/` в контейнер, затем запускаем модуль:

```bash
# поднять стек (если не запущен)
docker compose watch

# скопировать каталог в контейнер и залить товары
docker compose cp ./items backend:/app/items
docker compose exec backend python -m app.seed_items
```

Фотографии копируются в `STATIC_DIR/item-photo/...` и раздаются по `/static/...`.

### Флаги

| Флаг                | По умолчанию          | Назначение                                                       |
| ------------------- | --------------------- | ---------------------------------------------------------------- |
| `--items-dir PATH`  | `/app/items`          | Директория с товарами.                                           |
| `--owner EMAIL`     | первый суперюзер      | Владелец создаваемых товаров.                                    |
| `--reset`           | —                     | Удалить ранее загруженные сидом товары (по `title`) перед заливкой. |

```bash
# повторная заливка с заменой (например, после правок в item.json)
docker compose exec backend python -m app.seed_items --reset
```

## Поведение

- **Идемпотентно**: товар с уже существующим `title` пропускается. Повторный
  запуск без `--reset` ничего не дублирует.
- **Валидация до записи**: если в данных есть ошибки (нет файла фото, цена
  превышает лимит, пустой `title`/`images`), скрипт выводит список ошибок и
  ничего не загружает.
- **`--reset` безопасен для истории заказов**: при удалении товара его корзины и
  вишлисты удаляются каскадно, а в позициях заказов `item_id` обнуляется
  (`SET NULL`) — суммы и снапшоты в заказах сохраняются.
